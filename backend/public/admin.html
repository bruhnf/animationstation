Guest Welcome Credits

Number of free credits granted to a brand-new guest account the first time someone opens the app on a device (before they sign up). Set to 0 to give guests no free try-ons. Does not affect existing guests — only applies to newly created guest sessions.

 

Welcome Bonus Credits (join offer)

Free credits granted once when a real account verifies its email (the "Free Credits when you join" offer). Raise it for a limited-time promotion, lower it, or set it to 0 to discontinue the offer entirely. Takes effect immediately for newly verified accounts — no redeploy and no app rebuild. The app reads this value to show "Limited time offer — N Free Credits when you join" (and hides the offer when 0).

 

Referral Reward Credits

Credits granted to both the referrer and the new user when a referred account verifies its email. Set to 0 to disable the referral reward (codes still work, just no payout). Takes effect immediately — no redeploy or app rebuild.

 

Referral Anti-Farming Cap

Max number of rewarded referrals a single referrer can earn within the rolling window (30 days). Past the cap, the referrer's payout is withheld (the new user still gets their join bonus). Set to 0 for unlimited. Takes effect immediately.

 

Video Credit Cost

Credits charged to generate one AI video (image-to-video). Video costs more to generate than a still image, so this defaults higher than a try-on's 1 credit. Minimum 1. Takes effect immediately.

 

Try-On Throttle (Soft Queue)

Paces rapid-fire try-on submissions so a burst doesn't spike AI cost — it never rejects a try-on, it just places the extra ones in a short queue. Each user gets a free burst (no wait) within the rolling window; past that, each further submission steps down the delay ladder. Higher tiers get a bigger burst (faster queues for subscribers). Buying credits / subscribing instantly clears a user's queue position. Takes effect on the next submission; no redeploy.

 

Splash / Announcement Screen

When a splash image is published here, the mobile app shows it full-screen on launch (promotions, the app logo, service-degradation notices) and the user must tap OK to continue. From the second launch onward they can opt out of the current splash. Publishing a new image resets that — everyone sees it at least once. No image published = the app starts normally. Changes take effect immediately; no app rebuild or backend redeploy needed.
